# Data Safety Yes/No 매트릭스 (초안)

기준일: 2026-04-11
근거: 앱/백엔드 소스 코드 및 현재 릴리즈 구성

## 1) 상위 문항
- 데이터 수집 여부: Yes
- 데이터 공유 여부: No (현재 릴리즈 구성 기준)
- 전송 중 암호화: Yes (프로덕션 HTTPS 강제)

## 2) 데이터 유형별 초안
| 카테고리 | 항목 | 수집 | 공유 | 필수/선택 | 주요 목적 |
|---|---|---|---|---|---|
| 개인 정보 | 이름 | Yes | No | 필수 | 앱 기능 제공, 계정 관리 |
| 개인 정보 | 이메일 | Yes | No | 선택 | 계정/운영 연락 |
| 개인 정보 | 사용자 ID(사번/계정ID) | Yes | No | 필수 | 로그인/권한 제어 |
| 금융 정보 | 회비 납부 상태/이력 | Yes | No | 서비스 기능상 필요 | 회비 관리 기능 |
| 앱 활동 | 출석/리그/회비 관련 앱 내 활동 | Yes | No | 서비스 기능상 필요 | 앱 기능 제공 |

## 3) 검토 메모
- 광고 SDK, 분석 SDK의 활성 수집/공유 플로우는 현재 릴리즈 구성에서 확인되지 않음.
- android build 설정에 google-services classpath는 있으나, google-services.json 부재로 플러그인 미적용 상태.
- 사용자 셀프 삭제 기능은 확인되지 않았으므로 정책 문서에 삭제 요청 채널을 명시해야 함.

## 4) 제출 시 주의
- Play Console 문항은 최신 릴리즈 바이너리 기준으로 최종 확인 후 제출.
- 향후 Firebase/분석 SDK 추가 시 본 문서를 즉시 갱신.
