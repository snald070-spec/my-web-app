# Play Console 클릭 순서 런북 (내부 테스트 기준)

기준일: 2026-04-11
대상 아티팩트: frontend/android/app/build/outputs/bundle/release/app-release.aab

## 1) 릴리즈 업로드
1. Play Console 접속
2. 앱 선택
3. 테스트 > 내부 테스트
4. 새 릴리즈 만들기
5. app-release.aab 업로드
6. 릴리즈 노트 입력
7. 저장

## 2) 앱 콘텐츠 점검
1. 정책 > 앱 콘텐츠 이동
2. 앱 액세스 항목 열기
3. 로그인 필요 선택
4. 심사용 테스트 계정/접속 경로 입력
5. 저장

## 3) 데이터 세이프티 입력
1. 정책 > 앱 콘텐츠 > 데이터 세이프티
2. 데이터 수집: 예
3. 데이터 공유: 아니오(현재 릴리즈 구성 기준)
4. 전송 중 암호화: 예
5. 데이터 삭제 요청 방법: 개인정보처리방침 내 이메일/폼 안내 반영
6. 각 데이터 유형별 항목 입력 후 저장

참조: PLAY_DATA_SAFETY_MATRIX_KO.md

## 4) 개인정보처리방침 URL 입력
1. 스토어 설정 또는 앱 콘텐츠의 정책 입력 구간으로 이동
2. 공개 접근 가능한 개인정보처리방침 URL 입력
3. 저장

참조: PRIVACY_POLICY_KO_DRAFT.md

## 5) 콘텐츠 등급
1. 정책 > 앱 콘텐츠 > 콘텐츠 등급
2. 설문 응답
3. 저장

## 6) 내부 테스트 배포
1. 내부 테스트 트랙에서 검토/게시
2. 테스터 그룹 지정
3. 옵트인 링크 확인
4. 설치 가능 여부 확인

## 7) 제출 전 마지막 체크
- AAB 업로드 버전과 실제 코드가 일치하는지 확인
- 심사 계정 로그인 가능 여부 재검증
- 개인정보처리방침 URL 공개 접근 가능 여부 확인
- 데이터 세이프티 응답이 최신 릴리즈 구성과 일치하는지 확인
