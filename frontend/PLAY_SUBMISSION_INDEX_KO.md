# Play 제출 인덱스 (단일 진입점)

이 파일부터 순서대로 진행하면 내부 테스트 트랙 제출까지 완료할 수 있습니다.

## 1) 핵심 입력본
- Play Console 입력본: PLAY_CONSOLE_INPUT_KO_FINAL.md
- Data Safety 매트릭스: PLAY_DATA_SAFETY_MATRIX_KO.md
- 개인정보처리방침 초안: PRIVACY_POLICY_KO_DRAFT.md
- Placeholder 통합 입력 시트: PLAY_PLACEHOLDER_FILL_SHEET_KO.md
- 최종장(프로토타입 설치 파일 안내): FINAL_PROTOTYPE_HANDOFF_KO.md

## 2) 실제 작업 순서
1. PLAY_CONSOLE_CLICK_RUNBOOK_KO.md 열기
2. 내부 테스트 릴리즈 업로드
3. 앱 액세스 입력(심사용 계정 제공)
4. 데이터 세이프티 입력
5. 개인정보처리방침 URL 입력
6. 내부 테스트 게시

## 3) 템플릿 문서
- 심사용 계정 템플릿: PLAY_REVIEWER_ACCOUNT_TEMPLATE_KO.md
- 내부 테스트 릴리즈 노트: PLAY_INTERNAL_TEST_RELEASE_NOTES_KO.md
- 테스터 공유 안내문: TESTER_INVITE_MESSAGE_KO.md
- 테스트 피드백 폼: TEST_FEEDBACK_FORM_KO.md
- Android 설치 트러블슈팅: ANDROID_INSTALL_TROUBLESHOOTING_KO.md

## 4) 현재 상태 확인
- 사전 점검 현황: PLAY_PRECHECK_STATUS.md
- 제출 초안 근거: PLAY_CONSOLE_SUBMISSION_DRAFT.md

## 5) 제출 직전 필수 확인
- 개인정보처리방침 URL이 외부에서 접근 가능한지
- 심사용 계정으로 로그인 가능한지
- Data Safety 응답이 최신 빌드와 일치하는지
- 업로드한 AAB가 최신 아티팩트인지
